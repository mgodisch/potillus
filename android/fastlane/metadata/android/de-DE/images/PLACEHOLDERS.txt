PLACEHOLDER — store graphics (drop the real binaries here, then delete this file)
=================================================================================

This directory holds the visual assets that `fastlane supply` (Google Play) and
F-Droid read. None of the binaries can be auto-generated, so they are NOT shipped
in the repository. Add files with EXACTLY these names, sizes and formats so both
stores accept them without complaint:

icon.png
    Google Play hi-res icon. 512 x 512 px, 32-bit PNG (with alpha), <= 1 MB.
    The launcher icon in res/mipmap-* is only 192 x 192 and is therefore NOT a
    valid substitute — export a dedicated 512 x 512 version.

featureGraphic.png
    Google Play feature graphic. 1024 x 500 px, PNG or JPEG, no alpha.
    Required by Play; ignored by F-Droid if absent.

phoneScreenshots/
    2 to 8 screenshots. PNG or JPEG, no alpha. Each side 320–3840 px, with the
    longer side at most twice the shorter (Play). Name them in display order,
    e.g. 1.png, 2.png, 3.png … Portrait 9:16 is recommended for a phone app.

Optional (create the directory only if you supply assets):
    sevenInchScreenshots/   tenInchScreenshots/   tvScreenshots/
    wearScreenshots/        tvBanner.png          promoGraphic.png

Notes
-----
* Localized assets override the default. If an image is identical across
  languages you may instead place it once under en-US and leave the other
  locales without an images/ entry.
* Keep file names lowercase and exactly as above; the stores match by name.
